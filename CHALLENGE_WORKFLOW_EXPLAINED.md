# FIFA Skeletal Tracking Challenge - Complete Workflow

## 📋 Overview

The FIFA Skeletal Tracking Challenge requires you to predict **3D skeleton positions** for all players across multiple frames of soccer match footage. The challenge has two stages:

1. **Camera Calibration/Tracking** - Track camera movement across all frames
2. **3D Pose Estimation** - Predict 3D skeleton positions for each player

## 🎯 Challenge Goal

**Input:** Video sequences with:
- Initial camera parameters (frame 0 only)
- Player bounding boxes
- Camera intrinsics (K, k) per frame

**Output:** 3D skeleton positions (15 keypoints × 3 coords) for each player in each frame

**Submission Format:** `submission_val.zip` and `submission_test.zip` containing `.npz` files with 3D predictions

**What happens inside main.py:**

```python
def process_sequence(boxes, cameras, skels_3d, skels_2d, video_path):
    """
    For each sequence:
    1. Initialize camera tracker with R[0], t[0]
    2. For each frame:
       a. Track camera using optical flow
       b. Every 10 frames: refine with lane detection
       c. Estimate 3D positions for each player
       d. Optimize positions to minimize reprojection error
    3. Return: 3D predictions + tracked cameras
    """
    
    # Initialize
    camera_tracker = CameraTracker(pitch_points, fps=50.0)
    camera_tracker.initialize(K=K[0], k=k[0], R=R[0], t=t[0])
    
    Rt = []  # Store tracked (R, t) for each frame
    predictions = []  # Store 3D skeletons
    
    for frame_idx in range(NUM_FRAMES):
        # Read video frame
        img = video.read()
        
        # ========================================
        # CAMERA TRACKING (Part 1 of challenge)
        # ========================================
        
        # Track camera using:
        # - Optical flow: Track 714 pitch points across frames
        # - Lane detection: Refine rotation every 10 frames
        state = camera_tracker.track(frame_idx, img, K[frame_idx], k[frame_idx])
        
        Rt.append((state.R, state.t))  # Save tracked camera
        
        # ========================================
        # 3D POSE ESTIMATION (Part 2 of challenge)
        # ========================================
        
        for player_idx in range(NUM_PLAYERS):
            # Find lowest visible keypoint (foot on ground)
            lowest_2d = skel_2d[frame_idx, player_idx].argmax(axis=0)[1]
            
            # Cast ray from 2D pixel through camera
            origin, direction = ray_from_xy(lowest_2d, K, R, t, k1, k2)
            
            # Find intersection with pitch plane (Z=0)
            foot_3d = intersection_over_plane(origin, direction)
            
            # Transform skel_3d from camera-relative to world coords
            skel_3d_world = skel_3d[frame_idx, player_idx] @ R
            skel_3d_world -= skel_3d_world[lowest_idx]
            skel_3d_world += foot_3d
            
            predictions[player_idx, frame_idx] = skel_3d_world
        
        # Fine-tune: Optimize translation to minimize reprojection error
        predictions = optimize_positions(predictions, skel_2d, Rt, K, k)
    
    # Update camera dict with tracked poses
    cameras['R'] = np.array([R for R,t in Rt])  # (N,3,3)
    cameras['t'] = np.array([t for R,t in Rt])  # (N,3)
    
    return predictions  # (N_players, N_frames, 15, 3)
```

**Key Camera Tracking Methods:**

1. **Optical Flow (Every Frame)**
   ```python
   # Project pitch points using previous camera pose
   pts_prev_2d = project(pitch_points, R_prev, t_prev, K, k)
   
   # Track to current frame using Lucas-Kanade
   pts_curr_2d = cv2.calcOpticalFlowPyrLK(frame_prev, frame_curr, pts_prev_2d)
   
   # Estimate relative rotation from correspondences
   R_relative = estimate_rotation(pts_prev_2d, pts_curr_2d)
   R_curr = R_relative @ R_prev
   ```

2. **Lane Line Refinement (Every 10 Frames)**
   ```python
   # Detect white field markings
   mask = adaptive_threshold(frame)
   dist_map = distance_transform(mask)
   
   # Optimize: align projected pitch points with detected lines
   R_refined = optimize(R_curr, minimize=distance_to_lines)
   ```

## 🔧 How Camera Calibration Currently Works

### **Architecture: CameraTracker Class**

Located in `lib/camera_tracker.py`, this class maintains camera state and tracks it across frames.

**State Variables:**
```python
class CameraState:
    frame_idx: int
    K: np.ndarray   # (3,3) intrinsics
    k: np.ndarray   # (5,) distortion
    R: np.ndarray   # (3,3) rotation
    C: np.ndarray   # (3,) camera center
    
    @property
    def t(self):
        return -self.R @ self.C  # Translation vector
```

**Tracking Pipeline:**

```
Frame 0:
  Initialize: R[0], t[0] from cameras/*.npz ✓

Frame 1-N:
  ┌─────────────────────────────────────────┐
  │ 1. Optical Flow Tracking (every frame) │
  │    - Project 714 pitch points           │
  │    - Track pixels with Lucas-Kanade    │
  │    - Estimate rotation from motion      │
  └─────────────────────────────────────────┘
            ↓
  ┌─────────────────────────────────────────┐
  │ 2. Lane Detection (every 10 frames)    │
  │    - Extract white line mask           │
  │    - Create distance transform         │
  │    - Optimize rotation alignment       │
  └─────────────────────────────────────────┘
            ↓
  ┌─────────────────────────────────────────┐
  │ 3. Save tracked R[i], t[i]             │
  │    - Store in Rt array                 │
  │    - Used for 3D pose estimation       │
  └─────────────────────────────────────────┘
```

**Key Algorithms:**

1. **Optical Flow:**
   - Uses `cv2.calcOpticalFlowPyrLK` (Pyramidal Lucas-Kanade)
   - Tracks ~714 pitch points between consecutive frames
   - Robust to moderate camera motion
   - Filters outliers with modified Z-score

2. **Lane Detection:**
   - Adaptive thresholding to detect white lines
   - HSV color filtering to isolate field
   - Distance transform for spatial alignment
   - Uses `scipy.optimize` to minimize distance

3. **Rotation Estimation:**
   - Converts 2D flows to 3D rays (undistort points)
   - Solves for rotation using SVD
   - Ensures orthogonality with polar decomposition

## 💡 Key Insights

### **Why Camera Tracking is Critical**

**Without tracked cameras:**
- Frame 0: Accurate (has true R[0], t[0])
- Frame 100: Completely wrong (camera has moved!)

**With tracked cameras:**
- All frames: Accurate (has R[i], t[i] for all i)
- Proper 3D world coordinates
- Correct reprojection for optimization

### **What Makes This Challenging**

1. **Camera Motion:** Broadcast cameras pan, tilt, zoom during match
2. **Player Occlusion:** Players overlap, go off-screen
3. **World Coordinates:** skel_3d is camera-relative, needs transformation
4. **Optimization:** Balancing optical flow smoothness vs. accuracy

### **Baseline Performance**

The provided baseline is **intentionally simple** to get you started. Expected improvements:
- Better camera tracking (deep learning-based)
- Better 3D pose estimation (temporal models)
- Better reprojection optimization
- Multi-view consistency (if using multiple cameras)
